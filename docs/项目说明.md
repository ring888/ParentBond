# ParentBond 项目说明

## 1. 项目定位

ParentBond 是一个面向家庭亲子陪伴场景的 Web/PWA 应用。项目以孩子的每日任务、专注陪伴、成长记录和零花钱反馈为核心，提供孩子端与家长端两个视角：

- 孩子端：整理今日任务、番茄专注、完成反馈、小游戏训练、个人资料与成就。
- 家长端：查看孩子今日进度、参与陪伴专注、管理家庭成员、发起奖励或扣款、处理孩子申诉。
- AI 辅助：把口语化作业描述整理为结构化任务，支持 MiMo、OpenAI、Claude、DeepSeek，并带本地兜底解析。

当前项目是一个 npm workspaces 单仓库，包含 React 前端、NestJS 后端和共享类型包。

## 2. 技术栈

| 层级 | 技术 |
| --- | --- |
| 前端 | React 18、Vite、TypeScript、lucide-react、PWA manifest |
| 后端 | NestJS 10、TypeScript、TypeORM、class-validator |
| 数据库 | MySQL 8，字符集 utf8mb4 |
| AI 接入 | MiMo、OpenAI、Claude、DeepSeek |
| 包管理 | npm workspaces，Node.js >= 20 |
| 部署 | Docker、Docker Compose、Nginx 静态服务与反向代理 |

## 4. 核心模块

| 模块 | 后端位置 | 主要职责 |
| --- | --- | --- |
| health | `apps/api/src/modules/health` | 健康检查，接口为 `GET /api/v1/health` |
| auth | `apps/api/src/modules/auth` | 注册、登录、加入家庭、孩子加入、会话心跳与退出 |
| profile | `apps/api/src/modules/profile` | 孩子资料、家长资料、家长查看孩子详情 |
| tasks | `apps/api/src/modules/tasks` | 今日任务读取/保存、AI 解析、完成状态、SSE 实时任务流 |
| focus | `apps/api/src/modules/focus` | 专注记录、今日统计、亲子陪伴专注心跳 |
| games | `apps/api/src/modules/games` | 舒尔特、Stroop、N-back、反应力等小游戏记录与汇总 |
| wallet | `apps/api/src/modules/wallet` | 零花钱奖励/扣款、证据上传、孩子确认/申诉、家长复核 |
| ai | `apps/api/src/modules/ai` | 多模型任务解析与本地 fallback |

前端入口为 `apps/web/src/main.tsx`，主应用在 `apps/web/src/App.tsx`。认证页和会话守卫分别位于 `apps/web/src/components/AuthPage.tsx`、`apps/web/src/components/AuthGate.tsx`。

## 5. 数据库与文件存储

数据库使用 MySQL，初始化脚本位于：

```text
apps/api/src/database/schema.sql
```

后端 TypeORM 当前配置为 `synchronize: false`，也就是说表结构不会由 TypeORM 自动创建，首次部署时需要执行 `schema.sql`。Docker Compose 已把该 SQL 挂载到 MySQL 初始化目录，第一次创建数据库卷时会自动导入。

上传文件用于零花钱证据，后端以 `/uploads/` 公开访问。Docker 部署时上传目录使用名为 `api-uploads` 的 Docker volume 持久化。

## 6. 主要环境变量

| 变量 | 说明 | 示例 |
| --- | --- | --- |
| `PORT` | API 监听端口 | `3000` |
| `HOST` | API 监听地址 | `0.0.0.0` |
| `WEB_ORIGIN` | 允许跨域访问 API 的前端来源，多个用逗号分隔 | `http://localhost:8080` |
| `MYSQL_HOST` | MySQL 主机 | `mysql` |
| `MYSQL_PORT` | MySQL 端口 | `3306` |
| `MYSQL_USER` | MySQL 用户 | `parentbond` |
| `MYSQL_PASSWORD` | MySQL 密码 | 部署时自行设置 |
| `MYSQL_DATABASE` | MySQL 数据库名 | `parentbond` |
| `AI_DEFAULT_PROVIDER` | 默认 AI 解析服务 | `mimo` |
| `MIMO_API_KEY` | MiMo API Key | 可为空，空时任务解析走 fallback |
| `OPENAI_API_KEY` | OpenAI API Key | 可为空 |
| `CLAUDE_API_KEY` | Claude API Key | 可为空 |
| `DEEPSEEK_API_KEY` | DeepSeek API Key | 可为空 |
| `VITE_API_BASE_URL` | 前端构建时写入的 API 地址 | `http://localhost:8080/api/v1` |

注意：`VITE_API_BASE_URL` 是前端构建时变量，修改后需要重新构建 web 镜像。

## 7. 本地开发

准备 Node.js 20+、MySQL 8，并在项目根目录执行：

```powershell
npm install
Copy-Item apps/api/.env.example apps/api/.env
```

修改 `apps/api/.env` 中的数据库账号和 AI Key，然后在 MySQL 中创建数据库并导入：

```powershell
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS parentbond CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -u root -p parentbond < apps/api/src/database/schema.sql
```

启动前后端：

```powershell
npm run dev:api
npm run dev:web
```

默认访问：

- 前端：`http://localhost:5173`
- API 健康检查：`http://localhost:3000/api/v1/health`

生产构建验证：

```powershell
npm run build
```

## 9. 交付与部署注意事项

- 首次部署必须初始化数据库表；Docker Compose 会在 MySQL volume 第一次创建时自动执行 `schema.sql`。
- 修改数据库结构时，需要新增 SQL 迁移脚本或手动执行 ALTER。当前项目没有独立迁移系统。
- 上传文件保存在 Docker volume 中，重建 API 镜像不会丢失，但删除 volume 会丢失。
- AI Key 为空时，作业解析接口不会整体不可用，会走本地 fallback 逻辑。
- 前端 API 地址在构建时写入。换服务器 IP、域名或 HTTPS 地址后，需要重新构建 web 镜像。
- 正式环境建议使用 HTTPS，并把 `WEB_ORIGIN` 和 `VITE_API_BASE_URL` 改成真实域名。
